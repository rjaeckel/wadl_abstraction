http://localhost:8080/full/rest/api/company/createPOST

 This method provides create a company.
Only root of the platform is allowed to use this method.
Except for the id (which will be automatically set with company's name), all attributes of company are required. Mandatory attributes are:
  • A name (unique),
  • A postal address,
  • An email (respecting the format of an email and not already used); the email is automatically converted in lowercase letters.
  • And a phone number.
Also:
  • Root can register the company's companyProfile, a map of properties,
  • An applicationProfile is automatically created and attached to current company
Concerning the company's companyProfile, following entries can be specified (the default value is used for missing entries):
  • AUTO_PROVISIONING_ACCEPTED (default value is YES),
  • DEVICE_DISPLAYED (default value is YES),
  • SIM_DISPLAYED (default value is NO),
  • SECURITY_LEVEL (default value is BASIC_AUTHENTICATION),
  • MAX_STORED_EVENTS (default value is 10000) and
  • COMPANY_TOPIC
The returned company has an id and all previous attributes (except the applicationProfile).

http://localhost:8080/full/rest/api/gateways/{deviceId}/{serviceName}/{actionName}/subscribePOST

 This method provides subscribe to a service/action of a gateway.
Root of the platform can subscribe to any service/action of any gateway.
An admin/a user role can ONLY subscribe to a service/action of his company's gateway.
The input subscription structure (XML/JSON) is used:
  • id, it is ignored and automatically generated
  • date, it is ignored and automaticcally set to time of subscription creation
  • duration (in hour), if not specified (or if the given value is less than 1), it is set to -1 which means that the subscription has an infinite duration
  • status, it is ignored and set to PENDING by default
  • application-profile id, it is required and if it is unavailable, a WEBApplicationException is thrown
  • action, it is ignored and determined with path parameters (gatewayId, serviceName and actionName)
  • actionValues, represents the arguments to give to the target action
The status of the subscription is PENDING by default. Then, a SUBSCRIPTION message is sent to the gateway with the same duration as subscription and with actionValues given. When the RESPONSE of subscription from the gateway is received, the status became ACTIVATED and EVENT message from the gateway are sent to COMPANY_TOPIC defined in companyProfile. When currentDate is greater than subscriptionCreationDate + duration, the subscription status became DEACTIVATED (stopped).

http://localhost:8080/full/rest/api/gateways/{deviceId}/{serviceName}/{actionName}/subscribe

Methods
POST

This method provides subscribe to a service/action of a gateway.
Root of the platform can subscribe to any service/action of any gateway.
An admin/a user role can ONLY subscribe to a service/action of his company's gateway.
The input subscription structure (XML/JSON) is used:

  • id, it is ignored and automatically generated
  • date, it is ignored and automaticcally set to time of subscription creation
  • duration (in hour), if not specified (or if the given value is less than 1), it is set to -1 which means that the subscription has an infinite duration
  • status, it is ignored and set to PENDING by default
  • application-profile id, it is required and if it is unavailable, a WEBApplicationException is thrown
  • action, it is ignored and determined with path parameters (gatewayId, serviceName and actionName)
  • actionValues, represents the arguments to give to the target action
The status of the subscription is PENDING by default. Then, a SUBSCRIPTION message is sent to the gateway with the same duration as subscription and with actionValues given. When the RESPONSE of subscription from the gateway is received, the status became ACTIVATED and EVENT message from the gateway are sent to COMPANY_TOPIC defined in companyProfile. When currentDate is greater than subscriptionCreationDate + duration, the subscription status became DEACTIVATED (stopped).
