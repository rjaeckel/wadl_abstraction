//rest/api/company/create

Methods

POST

This method provides create a company.<br> Only root of the platform is allowed to use this method.<br> Except for the id (which will be automatically set with company's name), all attributes of company are required. Mandatory attributes are:<br> <ul> <li>A name (unique),</li> <li>A postal address,</li> <li>An email (respecting the format of an email and not already used); the email is automatically converted in lowercase letters.</li> <li>And a phone number.</li> </ul> Also:<br> <ul> <li>Root can register the company's companyProfile, a map of properties,</li> <li>An applicationProfile is automatically created and attached to current company</li> </ul> Concerning the company's companyProfile, following entries can be specified (the default value is used for missing entries):<br> <ul> <li>AUTO_PROVISIONING_ACCEPTED (default value is YES),</li> <li>DEVICE_DISPLAYED (default value is YES),</li> <li>SIM_DISPLAYED (default value is NO),</li> <li>SECURITY_LEVEL (default value is BASIC_AUTHENTICATION),</li> <li>MAX_STORED_EVENTS (default value is 10000) and</li> <li>COMPANY_TOPIC</li> </ul> The returned company has an id and all previous attributes (except the applicationProfile).<br>

acceptable request representations:

  • text/xml (company)
  • application/json

available response representations:

  • text/xml (company)
  • application/json

//rest/api/company/update

Methods

POST

acceptable request representations:

  • text/xml (company)
  • application/json

When the RESPONSE of subscription from the gateway is received, the status became ACTIVATED and EVENT message from the gateway are sent to COMPANY_TOPIC defined in companyProfile. When currentDate is greater than subscriptionCreationDate + duration, the subscription status became DEACTIVATED (stopped).
